# MEXC Triangular Arbitrage Bot

Бот для внутрибиржевого треугольного арбитража на бирже **MEXC** (спот).
Управление через Telegram (кнопки), мониторинг в терминале и Google Sheets.

> **Статус:** все 6 вех готовы (dry-run полностью рабочий; live — после прогона в dry и вставки ключей).
> Остальные вехи — в разработке (см. [Дорожную карту](#дорожная-карта)).

---

## Что делает бот

1. Получает актуальные спотовые пары MEXC и отбирает торгуемые.
2. Строит треугольные связки (`USDT → A → B → USDT`) в обоих направлениях (BBS / BSS).
3. Подключается по WebSocket к книгам ордеров нужных пар.
4. Считает спред тейкером на заданную сумму по реальной глубине стакана (VWAP).
5. Вычитает три комиссии.
6. Если net-спред выше порога — торгует (в live-режиме), иначе продолжает мониторинг.
7. Торгует как тейкер (MARKET / IOC / FOK).
8. Подводит итоги и продолжает мониторинг.

Есть безопасный **dry-run** режим (только сигналы в лог, без реальных ордеров) —
обязателен для старта, так как у MEXC нет тестнета.

## Архитектура

Разделение на «медленный» слой управления (Telegram) и «быстрый» торговый движок
(hot path). Telegram меняет только конфиг; движок читает его атомарно.

```
mexc_arb/
├── core/               # чистая логика (без сети): модели, треугольники, VWAP, спред
│   ├── models.py           # Pair, Leg, Triangle, Evaluation
│   ├── triangles.py        # граф рынка, построение связок, индекс symbol→треугольники
│   ├── vwap.py             # проход стакана тейкером (walk_buy / walk_sell)
│   └── spread.py           # оценка треугольника, вычет комиссий, сканирование
├── connectors/         # общение с MEXC
│   ├── auth.py             # подпись HMAC-SHA256
│   └── mexc_rest.py        # REST-клиент (публичные эндпоинты)
├── engine/             # движок
│   └── pair_selector.py    # отбор пар и построение треугольников
├── infra/              # инфраструктура
│   ├── logging_conf.py     # логи с тегами этапов
│   └── config.py           # конфиг (секреты из окружения)
├── reporting/          # отчётность (веха 4): терминал + Google Sheets
├── control/            # Telegram-пульт (веха 5)
├── scripts/            # демо и утилиты
│   └── demo_pair_selection.py
└── tests/fixtures/     # снапшоты для офлайн-тестов
```

## Требования

- Python 3.11+
- Ключи API MEXC (для живого режима)
- Сервисный аккаунт Google Cloud (для записи в Google Sheets, веха 4)
- Telegram-бот и токен (веха 5)

## Установка

```bash
git clone <URL вашего репозитория>
cd mexc_arb
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Настройка

```bash
cp .env.example .env
# затем заполните .env своими ключами (файл .env в .gitignore, не коммитится)
```

## Запуск демо (без сети, проверка логики)

```bash
python -m scripts.demo_pair_selection
```

Демо загружает снапшоты из `tests/fixtures/`, отбирает пары, строит треугольники
и проверяет расчёт спреда — с пошаговыми логами.

## Особенности MEXC (важно)

- Ордера ставятся **только через REST** (`POST /api/v3/order`) — WS-ордеров нет.
- WebSocket рыночных данных — в формате **Protocol Buffers** (нужна компиляция `.proto`).
- **Тестнета нет** — API сразу боевой, поэтому dry-run обязателен.
- Комиссии часто 0%, но не на всех парах — ставка тянется из данных биржи.
- Лимиты: ордера 12 req/s, WS ≤30 подписок на соединение, соединение ≤24ч.

Проект носит технический характер. Это не финансовый совет. Треугольный арбитраж —
высококонкурентная область; торговля сопряжена с риском потери капитала.
Оценка рисков — на пользователе.
